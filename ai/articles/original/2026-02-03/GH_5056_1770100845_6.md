üß± How It Works (Architecture)
------------------------------

[](http://github.com/ghuntley/how-to-build-a-coding-agent#-how-it-works-architecture)

Each agent works like this:

1.   Waits for your input
2.   Sends it to Claude
3.   Claude may respond directly or ask to use a tool
4.   The agent runs the tool (e.g., read a file)
5.   Sends the result back to Claude
6.   Claude gives you the final answer

We call this the **event loop** ‚Äî it's like the agent's heartbeat.

üí° How Tools Work (Under the Hood)
----------------------------------

[](http://github.com/ghuntley/how-to-build-a-coding-agent#-how-tools-work-under-the-hood)

Tools are like plugins. You define:

*   **Name** (e.g., `read_file`)
*   **Input Schema** (what info it needs)
*   **Function** (what it does)

Example tool definition in Go:

undefinedgo
var ToolDefinition = ToolDefinition{
    Name:        "read_file",
    Description: "Reads the contents of a file",
    InputSchema: GenerateSchema[ReadFileInput](),
    Function:    ReadFile,
}
undefined

Schema generation uses Go structs ‚Äî so it‚Äôs easy to define and reuse.
