Ona (formerly Gitpod) is re-launching its Open Source program
=============================================================

[#opensource](http://dev.to/t/opensource)[#ai](http://dev.to/t/ai)[#ona](http://dev.to/t/ona)[#agents](http://dev.to/t/agents)

[Gitpod](https://github.com/gitpod-io/gitpod) started as an open-source project. Long before ‚ÄúAI productivity‚Äù became a thing, the core problem we were trying to solve was simple:

**help contributors get productive without wasting maintainer time.**

Over the years, working closely with open-source maintainers using Gitpod‚Äôs Open Source plan, the same issues kept coming up:

*   PR backlogs grow faster than they can be reviewed
*   Maintainers spend large amounts of time onboarding contributors and answering setup questions
*   Reviewing changes often means reconstructing context instead of focusing on intent and correctness
*   And now, with AI tools everywhere, maintainers also have to sift through a growing volume of low-signal or poorly contextualized PRs

Recently, Gitpod evolved into [**Ona**](https://ona.com/). The product has grown, but the maintainer problems haven‚Äôt gone away.

That‚Äôs why we‚Äôve brought back the **Open Source plan**, now as the **Ona for Open Source program**.

### [](http://dev.to/siddhantkcode/ona-formerly-gitpod-is-re-launching-its-open-source-program-5d3c#whats-the-focus) What‚Äôs the focus?

This isn‚Äôt about adding more tools. It‚Äôs about reducing friction where it hurts most.

**Ona for Open Source** is designed to help:

*   **Maintainers** review PRs faster by spending less time reconstructing context and unblocking contributors
*   **Projects** keep backlogs manageable as contribution volume increases
*   **Contributors** start working with clearer expectations and fewer setup-related questions
*   **Teams** keep signal high even as AI-assisted contributions become more common

If you‚Äôre curious about the transition from Gitpod to Ona, here‚Äôs more context: [https://ona.com/stories/gitpod-is-now-ona](https://ona.com/stories/gitpod-is-now-ona)

And if you maintain (or contribute to) an open-source project and want to check out the program: **[https://ona.com/open-source](https://ona.com/open-source)**. You can get up to $200/month in free credits.

Open source survives because maintainers keep showing up. If we can reduce even a small part of that load, especially in an AI-heavy world, it‚Äôs worth doing.

Happy to hear feedback, particularly from maintainers on what still feels broken.
